# Project 04 — Highly Available Kubernetes Cluster with kubeadm

Bootstraps a production-style Kubernetes 1.30 cluster from scratch using kubeadm — 1 control-plane + 2 worker nodes — on local Ubuntu VMs provisioned by multipass.

## Architecture

```
┌─────────────────────────────────────────────────────┐
│  k8s-control (Ubuntu 22.04 VM, 2 CPU / 2 GB)        │
│  ┌────────────────────────────────────────────────┐  │
│  │ kube-apiserver  kube-scheduler                 │  │
│  │ kube-controller-manager  etcd                  │  │
│  │ Calico CNI (calico-node DaemonSet)             │  │
│  └────────────────────────────────────────────────┘  │
└───────────────┬─────────────────┬───────────────────┘
                │ kubeadm join    │ kubeadm join
┌───────────────▼──────┐  ┌───────▼──────────────────┐
│  k8s-worker-1        │  │  k8s-worker-2             │
│  containerd + kubelet│  │  containerd + kubelet     │
│  Calico node agent   │  │  Calico node agent        │
└──────────────────────┘  └───────────────────────────┘

Container runtime : containerd 1.7 (SystemdCgroup = true)
CNI               : Calico 3.28  (podCIDR 192.168.0.0/16)
State store       : etcd (single, on control-plane)
```

## Prerequisites

| Tool | Install |
|------|---------|
| multipass | auto-installed by setup.sh via `snap` |
| kubectl | `snap install kubectl --classic` |

> Needs ~6 GB RAM and ~30 GB disk free for the three VMs.

## Quick Start

```bash
chmod +x setup.sh && ./setup.sh
```

After completion:
```bash
export KUBECONFIG=$(pwd)/kubeconfig
kubectl get nodes -o wide
kubectl get pods -A
```

## What Each Script Does

| Script | Purpose |
|--------|---------|
| `scripts/common.sh` | Runs on every node: disables swap, loads kernel modules, installs containerd (SystemdCgroup=true), installs kubeadm/kubelet/kubectl |
| `scripts/controlplane.sh` | Runs kubeadm init with a kubeadm-config.yaml, installs Calico CNI, waits for control-plane Ready |
| `scripts/worker.sh` | Runs the kubeadm join command generated by the control-plane |

## Key Interview Points

**Why disable swap?**
kubelet refuses to start if swap is on — memory limits become unpredictable when the kernel can swap pod memory out.

**Why `SystemdCgroup = true`?**
containerd and kubelet must use the same cgroup driver. On Ubuntu 22.04, systemd manages cgroups. Mismatch causes pods to fail to start.

**Why Calico?**
Calico implements the CNI spec with a BGP-based routing model. It also provides NetworkPolicy enforcement, which bare kubeadm clusters lack without a CNI.

**Why 192.168.0.0/16 for pods?**
It's Calico's default `CALICO_IPV4POOL_CIDR`. Must not overlap with node or service CIDRs.

**HA considerations (production)**
For true HA you need 3 control-plane nodes (for etcd quorum) behind a load balancer. kubeadm supports this via `--control-plane-endpoint` and `--upload-certs`. This project uses a single control plane for simplicity.

## Demo: Node Failure and Recovery

```bash
export KUBECONFIG=$(pwd)/kubeconfig

# Kill a worker
multipass stop k8s-worker-1

# Watch it go NotReady (~40s)
kubectl get nodes -w

# Pods reschedule to the remaining worker
kubectl get pods -o wide

# Bring it back
multipass start k8s-worker-1
kubectl get nodes -w   # returns to Ready
```

## Cleanup

```bash
multipass delete k8s-control k8s-worker-1 k8s-worker-2
multipass purge
rm -f kubeconfig
```
